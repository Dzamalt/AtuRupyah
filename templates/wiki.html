Getting Started
Everything you need to make your first request in minutes.

Prerequisites

  • All requests and responses use JSON format
  • Include Content-Type: application/json on all requests
  • Protected routes require a Bearer JWT token
Typical Workflow
1

Register a new user account via /auth/register

2

Authenticate your credentials to receive a JWT access token

3

Pass the token in the Authorization: Bearer <token> header

4

Manage products, inventory, sales, and view forecasts

Authentication
AtuRupyah uses Flask JWT. Authenticate once, then attach the token to every protected request.
Login Request
POST /auth/login
// Request body
{
  "username": "your_username",
  "password": "your_password"
}

// Response
{
  "access_token": "eyJhbGciOiJIUzI1NiIs..."
}
Using the Token
Authorization Header
Authorization: Bearer <access_token>

Auth
Register and authenticate users to receive JWT tokens.
POST /auth/register Register a new user account ▾

Creates a new user account. No authentication required.

Request Body
{
  "username": "string",
  "password": "string"
}
POST /auth/login Authenticate and receive JWT token ▾

Authenticate with your credentials. Returns an access token for subsequent requests.

Request Body
{
  "username": "your_username",
  "password": "your_password"
}
Response 200
{
  "access_token": "eyJhbGci..."
}
Products
Full CRUD operations for managing your product catalog.
GET /products Retrieve all products ▾

Returns a list of all products in the system.

🔒 Requires JWT
GET /products/{id} Retrieve a specific product ▾

Returns details for a single product by its ID.

🔒 Requires JWT
POST /products Create a new product ▾

Adds a new product to the catalog.

🔒 Requires JWT
Request Body
{
  "name": "string",
  "description": "string",
  "price": number
}
PUT /products/{id} Update a product ▾

Fully replaces the product record for the given ID.

🔒 Requires JWT
DELETE /products/{id} Delete a product ▾

Permanently removes the product from the catalog.

🔒 Requires JWT
Inventory
Monitor and manage stock levels with low-stock alert support.
GET /inventory/{product_id} View inventory levels ▾

Returns current stock quantity and any low-stock alerts for the given product.

🔒 Requires JWT
PUT /inventory/{product_id} Update product inventory ▾

Sets or replaces the inventory quantity for a product.

🔒 Requires JWT
Request Body
{
  "quantity": number,
  "reorder_level": number
}
PATCH /inventory/{product_id} Restock product inventory ▾

Adds stock to an existing inventory record — useful for partial restocking operations.

🔒 Requires JWT
Request Body
{
  "quantity": number
}
Sales
Track and record sale transactions across your product catalog.
GET /sales Retrieve all sales history ▾

Returns the complete sales transaction history across all products.

🔒 Requires JWT
GET /sales/{product_id} Retrieve sales for a product ▾

Returns sales history filtered to a specific product.

🔒 Requires JWT
POST /sales/{product_id} Record a sale transaction ▾

Logs a new sale for the given product and decrements inventory accordingly.

🔒 Requires JWT
Request Body
{
  "quantity_sold": number
}
Forecasts
Demand forecasting using moving averages with a full dashboard overview and restock recommendations.
GET /forecasts/{product_id} Get demand forecast ▾

Returns a moving average demand forecast for the specified product based on historical sales data.

🔒 Requires JWT
GET /forecasts/dashboard Get all products overview ▾

Returns a full dashboard view of all products including inventory status, sales trends, and restock recommendations.

🔒 Requires JWT
